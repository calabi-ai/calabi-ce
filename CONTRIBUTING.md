# Contributing to Calabi

Thank you for your interest in contributing to Calabi. We welcome contributions from the community.

## How to Contribute

### Reporting Bugs

- Use the [Bug Report](https://github.com/calabi-ai/calabi-ce/issues/new?template=bug_report.md) template
- Include your Docker version, OS, and Calabi version
- Attach relevant logs from `docker compose logs`

### Suggesting Features

- Use the [Feature Request](https://github.com/calabi-ai/calabi-ce/issues/new?template=feature_request.md) template
- Describe the use case and expected behavior

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Test your changes with `docker compose up -d`
4. Commit your changes with a clear message
5. Push to your fork and open a Pull Request

### What You Can Contribute

- Documentation improvements
- Docker Compose configuration enhancements
- Nginx gateway configuration
- Bug fixes in setup scripts
- Sample data improvements

### What Is Not Open for Contribution

The Calabi UI, license proxy, and branded images are proprietary. Contributions to these components are not accepted through this repository.

## Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## Questions?

Open a [GitHub Issue](https://github.com/calabi-ai/calabi-ce/issues) or reach out at support@calabi.dev.
